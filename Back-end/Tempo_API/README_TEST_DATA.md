# Тестовые данные для базы данных

Этот файл содержит инструкции по заполнению базы данных тестовыми данными.

## Использование

### Вариант 1: Через psql (командная строка)

```bash
# Windows (PowerShell)
psql -U your_username -d tempo_restaurant -f test_data.sql

# Linux/Mac
psql -U your_username -d tempo_restaurant -f test_data.sql
```

### Вариант 2: Через pgAdmin или другой GUI клиент

1. Откройте pgAdmin или другой клиент PostgreSQL
2. Подключитесь к базе данных `tempo_restaurant`
3. Откройте файл `test_data.sql`
4. Выполните весь скрипт

### Вариант 3: Через .NET приложение (опционально)

Можно создать отдельную команду или скрипт для заполнения данных через Entity Framework.

## Что содержит тестовый набор данных

### Пользователи для входа:
- **Администратор**: `admin` / `admin123`
- **Официант 1**: `waiter1` / `waiter123`
- **Официант 2**: `waiter2` / `waiter123`
- **Повар 1**: `cook1` / `cook123`
- **Повар 2**: `cook2` / `cook123`

### Данные включают:
- **5 категорий** (Горячие блюда, Холодные закуски, Десерты, Напитки, Салаты)
- **5 сотрудников** (1 админ, 2 официанта, 2 повара)
- **5 столов** (разной вместимости)
- **3 клиента**
- **10 ингредиентов**
- **4 вида посуды для готовки**
- **4 вида столовых приборов**
- **8 блюд** (борщ, стейк, курица, салаты, десерты)
- **6 напитков** (кола, соки, кофе, чай, вода)
- **4 заказа** (с разными статусами)
- **1 счет**

### Статусы заказов:
- `0` - Ordered (Заказан)
- `1` - Cooking (Готовится)
- `2` - Delivering (Доставляется)
- `3` - Ready (Готов)

## Очистка данных (опционально)

Если нужно очистить данные перед вставкой, раскомментируйте строку в начале файла `test_data.sql`:

```sql
TRUNCATE TABLE "Bill", "DrinkOrder", "DishOrder", "Order", "TablewareDish", "DishwareDish", "IngredientDish", "Drink", "Dish", "Tableware", "Dishware", "Ingredient", "Table", "Waiter", "Cook", "Employee", "User", "Category" CASCADE;
```

**Внимание:** Это удалит все существующие данные из указанных таблиц!

## Проверка данных

После выполнения скрипта вы увидите количество записей в каждой таблице. Убедитесь, что все данные вставлены корректно.

## Примечания

- Все UUID генерируются вручную для удобства тестирования
- Даты создания и обновления устанавливаются автоматически (NOW())
- Фото блюд и напитков используют примерные URL (замените на реальные при необходимости)
- Пароли хранятся в открытом виде (для тестирования; в продакшене используйте хеширование)
